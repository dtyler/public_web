What makes Vim annoying powerful: Modes

  • Modes allow for for reuse of keys, decreasing the need for keyboard-cording
  • The 4 modes in Vim are:
    • Normal
    • Insert
    • Visual
    • Command-Line

Mention mode prefixes for help search: i_, v_

Basic operations

  • (i)nsert text
  • (Esc)-ing out of where ever you are into normal mode
  • (:) - Enter command-line mode
  • (hjkl) - moves cursor Left, Down, Up, Right
  • :wq - Save and quit (followed by a file name if not yet named) (ZZ for short if already named)
  • :q - quits (ZQ for short. WARNING: This will not prompt you if you have unsaved work.)

Better movement

  • Searching
    • / - starts a search
    • n - next match
    • N - next match, searching backwards
  • Previous/Next Word
    • w - next word (start)
    • e - next word (end)
    • b - previous word (start)
    • ge - previous word (end)

Larger jumps

  • At times, you may want to repeat a command multiple times in a row.
  • e.g. Jump multiple words with 'w' or multiple lines with 'j'.
  • To do this, prefix the command with the number of times you'd like it repeated.

More Movement Keys
  • 0 - beginning of line
  • $ - end of line
  • gg - beginning of file
  • G - end of file
  • { - beginning of paragraph
  • } - end of paragraph
  • % - Jump to matching (),[],{}, or <>
  • Talk about default register for copy/paste
  • special movement commands that write to "1
    • %, (, ), `, /, ?, n, N, { and }

Registers

Registers are places to store text for later. There are a number of different kinds of registers

  • Numerical
    • "0: holds the next text most recently deleted or yanked
    • "1: holds old contents of register 0 after a delete (unless certain movement commands are used.)
    • "2-"9: with each delete, vim shifts old content along these registers
  • Named Registers: "a-"z
  • Commands with remove text from the screen (as well as the 'yank' command) can be prefixed with a register
  • You can view the contents of your registers with :registers
Tip: You can use CTRL-r in insert or command-line mode to insert the contents of a 
register at cursor postion

Macros

Executing the contents of registers

  • 'q' (followed by a register) begins recording commands into the given register (press 'q' again to finish)
  • '@' (followed by a register) plays out the contents of a register

Macros are often used for repeated actions:

qwI<li>^[A</li>^[jq
" The above sequence of keystrokes will wrap one line in <li> 
" and record the action to register 'w' (picked arbitrarily for 'wrap')

9@w 
" Runs the macro 9 times, wrapping a total of 10 lines 
" (including the one from the macro definition)

Macros - continued...

  • Breaking down the macro: qwI<li>^[A</li>^[jq
    • qw - record macro to register 'w'
    • I<li> - enter insert mode at the beginning of the line and type <li>
    • ^[ - return to normal mode
    • A</li> - enter insert mode at the end of the line and close the tag
    • ^[ - return to normal mode
    • j - Move the cursor down to the next line so that it is in the correct postion for 'chaining'
    • q - stop recording

Completions work across all open buffers and through c-style includes

Completions

  • i_CTRL-n - Offers completions for the current partial word under the cursor
  • i_CTRL-x - Enters detained completions mode, the next key hit will determine the kind of completions offered
    • CTRL-n | CTRL-p - Completion of words looking forward/backward respectively
    • CTRL-l - Completion on entire line
    • CTRL-k - dictionary completions (only works if spell checking is active)
    • CTRL-f - filename completion (both relative to current working directory and absolute paths work)

There is an option for controlling how new buffers will be opened (with respect to splits and tabs) called 'switchbuff'

Controlling Layout

  • Buffers
    • Every file opened in Vim is represented by a buffer
    • Buffers are also used to represent generated content like directory listings (when Vim is used to open a file) and the help system
    • Buffers can be associated with 0,1, or more "viewports" (my term, not Vim's)
  • Splits
    • Splits divide the screen into regions providing new space for displaying buffers.
    • They are created by splitting the current buffer either vertically (:vsplit) or horizontally (:split). Each of these command can take an optional buffer to display in the new split
  • Tabs
    • Each tab has it's own split layout
  • nocompatible - stop acting like vi
  • backspace=2 - short for: backspace=indent,eol,start. Allows for backspacing over automatically inserted indents, newlines, and the start of insert mode
  • laststatus - ensures that status bar is always shown

.vimrc - Tweaking Vim to Your Liking

  • Options
    • syntax on
    • set nocompatible
    • set smartindent
    • set expandtab
    • set backspace=2
  • Tab Characters
    • set softtabstop=4
    • set shiftwidth=4
  • Style
    • set number
    • set showcmd
    • set showmatch
    • set incsearch
    • set laststatus=2
    • set scrolloff=2
  • Keybindings
    • imap jj <ESC>
    • nnoremap ; :
    • vnoremap ; :
    • nmap <F12> :set number!<CR>
    • nmap <F4> :set invhls<CR>
    • nmap <F3> :set paste!<CR>

Misc

  • Use '.' to repeat your last action from current cursor position
  • Use CTRL-T/CTRL-D in insert mode to increase or decrease the indent of the current line
  • Explore the :colorscheme setting to color Vim to your liking
  • Vim can be further customized through plugins! If you think you'll be exploring this a lot, start by looking for a plugin managing plugin.
